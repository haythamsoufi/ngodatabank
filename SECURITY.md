# Security policy

## Supported versions

| Branch / scope | Supported          |
| -------------- | ------------------ |
| `main`         | Yes                |
| Other branches | Best effort        |

## Reporting a vulnerability

**Preferred:** use [GitHub Security advisories](https://github.com/haythamsoufi/humdatabank/security/advisories/new) to report issues **privately** (recommended for public repositories).

You may also contact: **haythamsoufi@outlook.com**.

Please include:

- Description of the issue and impact
- Steps to reproduce (or proof-of-concept) if possible
- Affected components (Backoffice, Website, MobileApp, infrastructure)

We will acknowledge receipt as soon as practical and coordinate disclosure.

## Out of scope (examples)

- Spam, social engineering, or physical security issues
- Reports from automated scanners without a clear, exploitable finding
- Issues in third-party dependencies without a practical impact on this repo’s usage (report upstream when appropriate)
