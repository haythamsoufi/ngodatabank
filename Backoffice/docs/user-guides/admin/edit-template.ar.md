# تحرير قالب نموذج (منشئ النماذج)

استخدم هذا الدليل عندما تحتاج إلى تحديث قالب موجود (الأسئلة والأقسام التي ستملأها نقاط الاتصال).

## قبل أن تبدأ

- تحتاج إلى وصول **المسؤول** مع أذونات القالب.
- تأكد مما إذا كان القالب قيد الاستخدام بالفعل (لديه مهام نشطة).
- حدد نوع التغيير الذي تحتاجه:
  - **آمن**: إصلاحات التسمية/نص المساعدة، إضافة حقول اختيارية، إعادة ترتيب الأقسام.
  - **محفوف بالمخاطر**: تغيير أنواع الحقول، تغيير قواعد المطلوب، حذف الحقول (يمكن أن يكسر الاتساق عبر الدول).

## فتح القالب

1. افتح **لوحة المسؤول** → **إدارة النماذج والبيانات** → **إدارة القوالب**.
2. ابحث عن القالب وافتحه.
3. إذا أظهر النظام إصدارات متعددة (مسودة/منشور)، ابدأ من **أحدث مسودة** (أو أنشئ إصدار مسودة جديد إن لزم الأمر).

## التحريرات الشائعة (خطوة بخطوة)

### إضافة قسم جديد

1. انقر على **إضافة قسم**.
2. أعط القسم اسماً واضحاً (يصبح هذا عنصر تنقل لنقاط الاتصال).
3. احفظ.

### إضافة حقل جديد

1. افتح القسم الذي تريد الحقل فيه.
2. انقر على **إضافة حقل**.
3. اختر نوع الحقل (نص/رقم/تاريخ/قائمة منسدلة، إلخ).
4. حدد **التسمية** (بالضبط ما سيراه المستخدمون).
5. إن كان متاحاً، أضف **نص مساعدة** (جملة قصيرة واحدة).
6. حدد **مطلوب** فقط عند الحاجة الحقيقية.
7. احفظ.

### تحديث تسمية أو نص مساعدة

1. افتح الحقل.
2. غيّر التسمية/نص المساعدة لتطابق المعنى الذي تريده.
3. احفظ.

### إعادة ترتيب الأقسام أو الحقول

1. استخدم مقبض السحب (أو عناصر التحكم في النقل) لإعادة الترتيب.
2. احفظ.

## أنواع الحقول (ما يجب اختياره)

- **النص**: الأسماء، الملاحظات، الشروحات القصيرة.
- **الرقم**: العدد والكميات. يُفضل الرقم عندما تحتاج إلى المجاميع أو التحقق.
- **التاريخ**: التواريخ (ليس نصاً حراً).
- **القائمة المنسدلة / اختيار واحد**: عندما يجب أن تكون الإجابات متسقة عبر الدول.
- **اختيار متعدد**: عندما قد تنطبق إجابات متعددة.
- **المصفوفة / الجدول** (إن كان متاحاً): قيم متكررة عبر الفئات. استخدم عندما يجب على المستخدمين إدخال نفس المقياس لصفوف متعددة.

## التحقق والحقول المطلوبة

- حدد حقل **مطلوب** فقط عندما يجب أن يكون موجوداً لكي يكون الإرسال قابلاً للاستخدام.
- إذا كان المستخدمون غالباً ما يُحجَزون عند **الإرسال**، قلل الحقول المطلوبة أو أضف تعليمات أوضح.
- عندما تغير قواعد التحقق، اختبر التأثير بمهمة صغيرة أولاً.

## ربط بنك المؤشرات (عند التطبيق)

إذا كان سير العمل يستخدم **بنك المؤشرات**:

- اربط حقل بمؤشر عندما تحتاج إلى تعريفات موحدة وإبلاغ متسق.
- حافظ على تسمية الحقل سهلة الاستخدام، حتى لو كان اسم المؤشر تقنياً.
- تجنب ربط أسئلة مختلفة متعددة بنفس المؤشر ما لم تمثل حقاً نفس المقياس.

## النشر والاختبار

1. احفظ تغييرات المسودة الخاصة بك.
2. إذا كان النظام يتطلب النشر، **انشر** الإصدار الجديد.
3. أنشئ مهمة اختبار صغيرة (دولة واحدة) وأكملها بنفسك.
4. أصلح أي تسميات مربكة وأزل الحقول المطلوبة غير الضرورية.

## المشاكل الشائعة

- **تغييراتي لا تظهر لنقاط الاتصال**: قد لا يكون إصدار القالب منشوراً أو قد تكون المهمة تستخدم إصداراً منشوراً أقدم.
- **المستخدمون لا يمكنهم الإرسال بعد تغييري**: حقل مطلوب أو تحقق جديد يحجب الإرسال—اختبر النموذج كنقطة اتصال.
- **البيانات تبدو غير متسقة عبر الدول**: تجنب تغيير أنواع الحقول/المعنى أثناء الجمع؛ أنشئ قالب/إصدار جديد واتصل بالتغيير.
