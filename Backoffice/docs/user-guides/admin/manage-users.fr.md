# Gérer les utilisateurs

Utilisez ce guide pour trouver des utilisateurs, mettre à jour leurs détails et gérer l'accès.

## Avant de commencer

- Vous avez besoin d'un accès **Administrateur**.

## Trouver un utilisateur

1. Ouvrez **Panneau d'administration** → **Gestion des utilisateurs** → **Gérer les utilisateurs**.
2. Utilisez la boîte de recherche pour trouver l'utilisateur par **nom** ou **e-mail**.
3. Ouvrez l'utilisateur pour voir ou modifier les détails.

## Mettre à jour les détails de l'utilisateur

1. Ouvrez l'utilisateur.
2. Dans l'onglet **Détails de l'utilisateur**, mettez à jour les champs dont vous avez besoin (par exemple : nom, titre, mot de passe).
3. Pour mettre à jour les rôles, sélectionnez ou désélectionnez les rôles dans la section Rôles (les utilisateurs peuvent avoir plusieurs rôles).
4. Cliquez sur **Enregistrer les modifications**.

## Assigner ou supprimer des pays/organisations (permissions d'entité)

1. Ouvrez l'utilisateur.
2. Allez dans l'onglet **Permissions d'entité**.
3. Ajoutez ou supprimez des **pays** ou des **organisations** (branches NS, divisions, etc.).
4. Cliquez sur **Enregistrer les modifications**.

## Désactiver un utilisateur

1. Trouvez l'utilisateur dans la liste.
2. Cliquez sur **Désactiver** (ou basculez le statut de l'utilisateur).
3. Confirmez.

Les utilisateurs désactivés ne peuvent pas se connecter, mais leur historique est préservé.

## Réinitialiser un mot de passe

Si votre flux de travail nécessite de définir un nouveau mot de passe pour un utilisateur :

1. Ouvrez l'utilisateur.
2. Définissez un nouveau mot de passe.
3. Enregistrez.
4. Partagez le nouveau mot de passe **de manière sécurisée**.

## Conseils

- Si un point focal signale des données manquantes, vérifiez d'abord les **assignations de pays**.
- Préférez **désactiver** les comptes au lieu de les supprimer.

## Problèmes courants

- **L'utilisateur ne peut pas voir les missions** : confirmez qu'il est un point focal et qu'il a au moins un pays assigné.
- **L'utilisateur ne peut pas accéder à une page d'administration** : confirmez que son rôle/permissions inclut cette page.
