# Notifications and communications (admin)

Use this guide to communicate clearly during a data collection cycle (reminders, instructions, and handling questions).

## Before you start

- Agree internally on one “source of truth” message (period, deadline, contact person).
- Keep communication short and actionable.

## What to communicate (recommended minimum)

For every assignment, communicate:
- **What** is requested (one sentence)
- **Who** should respond (countries/organizations)
- **When** it is due (deadline + time zone if needed)
- **How** to get help (one contact channel)

## Reminder cadence (simple approach)

Suggested cadence for a 2–4 week window:
- Reminder 1: shortly after launch (ensure everyone saw it)
- Reminder 2: mid-window (progress check)
- Reminder 3: 3–5 days before deadline (final push)
- Reminder 4: on deadline day (if appropriate)

## If users report issues (triage checklist)

Ask for:
- assignment name
- country/organization
- the exact step they were on (what button/page)
- screenshot of the error (if any)

Then route them to:
- [Troubleshooting basics (everyone)](../common/troubleshooting-basics.md)
- [Troubleshooting submission issues (Focal point)](../focal-point/troubleshooting-submission.md)
- [Troubleshooting templates and assignments (admin)](troubleshooting-templates-and-assignments.md)

## Best practices

- Use the exact UI labels users see (e.g. “Submit”, “Save draft”).
- Avoid long paragraphs; use bullet points.
- If you change anything mid-cycle (deadline, template wording), announce it clearly and record the decision.
