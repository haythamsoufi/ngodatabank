# Notificaciones y comunicaciones (administrador)

Usa esta guía para comunicarte claramente durante un ciclo de recopilación de datos (recordatorios, instrucciones y manejo de preguntas).

## Antes de comenzar

- Acuerda internamente un mensaje "fuente de verdad" (período, fecha límite, persona de contacto).
- Mantén la comunicación corta y accionable.

## Qué comunicar (mínimo recomendado)

Para cada asignación, comunica:
- **Qué** se solicita (una oración)
- **Quién** debe responder (países/organizaciones)
- **Cuándo** vence (fecha límite + zona horaria si es necesario)
- **Cómo** obtener ayuda (un canal de contacto)

## Ritmo de recordatorios (enfoque simple)

Ritmo sugerido para una ventana de 2-4 semanas:
- Recordatorio 1: poco después del lanzamiento (asegúrate de que todos lo vieron)
- Recordatorio 2: mitad de ventana (verificación de progreso)
- Recordatorio 3: 3-5 días antes de la fecha límite (último impulso)
- Recordatorio 4: el día de la fecha límite (si es apropiado)

## Si los usuarios reportan problemas (lista de verificación de triaje)

Pide:
- nombre de asignación
- país/organización
- el paso exacto en el que estaban (qué botón/página)
- captura de pantalla del error (si hay alguna)

Luego dirígelos a:
- [Conceptos básicos de solución de problemas (todos)](../common/troubleshooting-basics.md)
- [Solución de problemas de envío (Punto focal)](../focal-point/troubleshooting-submission.md)
- [Solución de problemas de plantillas y asignaciones (administrador)](troubleshooting-templates-and-assignments.md)

## Mejores prácticas

- Usa las etiquetas exactas de la interfaz que los usuarios ven (por ejemplo "Enviar", "Guardar borrador").
- Evita párrafos largos; usa viñetas.
- Si cambias algo a mitad del ciclo (fecha límite, redacción de plantilla), anúncialo claramente y registra la decisión.
