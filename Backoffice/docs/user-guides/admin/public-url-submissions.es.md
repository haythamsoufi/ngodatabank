# Envíos de URL públicas

Usa esta guía cuando quieras recopilar envíos sin requerir que los usuarios inicien sesión.

## Antes de comenzar

- Necesitas acceso de **Administrador** para gestionar asignaciones.
- Entiende el riesgo: las URL públicas pueden compartirse ampliamente. Úsalas solo cuando sea apropiado.

## Cuándo usar (y cuándo no)

Buenos casos de uso:
- Debes recopilar datos de colaboradores que no pueden obtener cuentas a tiempo.
- Necesitas un canal de captura de datos a corto plazo durante un evento urgente.

Evita cuando:
- Estás recopilando datos sensibles.
- Necesitas responsabilidad estricta (quién envió qué).

## Crear o habilitar una URL pública

1. Abre **Panel de administración** → **Gestión de formularios y datos** → **Gestionar asignaciones**.
2. Encuentra la asignación.
3. Si la asignación aún no tiene URL pública, haz clic en **Generar URL pública**.
4. Usa el interruptor para **activar** la URL pública.
5. Haz clic en **Copiar URL** para compartirla.

## Lista de verificación de gobernanza (recomendado)

Antes de compartir el enlace:
- Decide quién tiene permitido enviar (y a través de qué canal distribuirás la URL).
- Decide cómo manejarás **duplicados** (mantener el más reciente / mantener el mejor / revisión manual).
- Decide qué significa "calidad mínima" (campos requeridos, documentos requeridos, evidencia).
- Decide cuándo deshabilitarás el enlace (típicamente justo después de la fecha límite).

## Monitorear envíos públicos

1. Desde **Gestionar asignaciones**, usa:
   - **Ver todos los envíos públicos**, o
   - **Ver envíos** para una asignación específica.
2. Revisa cada envío por completitud y calidad.
3. Usa las acciones disponibles (por ejemplo aprobar/rechazar) para gestionar el control de calidad.

## Mejores prácticas

- Comparte la URL solo a través de canales de confianza (listas de correo que controlas, plataformas oficiales).
- Agrega instrucciones claras en el mensaje de asignación (qué completar, fecha límite, a quién contactar).
- Monitorea los envíos regularmente durante el período de recopilación.
- Decide cómo manejar duplicados (por ejemplo: mantener el más reciente, o mantener el que tiene las mejores fuentes).
- Deshabilita la URL pública cuando termine la ventana de recopilación (para reducir el riesgo).
- Si sospechas que el enlace se compartió demasiado ampliamente, deshabilítalo inmediatamente y crea un nuevo canal controlado.

## Problemas comunes

- **La URL pública "no funciona"**: confirma que está activada y que la asignación sigue activa.
- **Demasiados envíos incorrectos**: agrega instrucciones más claras, reduce la ambigüedad y considera deshabilitar la URL pública.
- **Envíos duplicados**: decide tu regla (mantener el más reciente / mantener el mejor / fusión manual) y aplícala consistentemente.
