# Public URL submissions

Use this guide when you want to collect submissions without requiring users to log in.

## Before you start

- You need **Admin** access to manage assignments.
- Understand the risk: public URLs can be shared widely. Use them only when appropriate.

## When to use (and when not to)

Good use cases:
- You must collect data from contributors who cannot get accounts in time.
- You need a short-term data capture channel during an urgent event.

Avoid when:
- You are collecting sensitive data.
- You need strict accountability (who submitted what).

## Create or enable a public URL

1. Open **Admin Panel** → **Form & Data Management** → **Manage Assignments**.
2. Find the assignment.
3. If the assignment has no public URL yet, click **Generate Public URL**.
4. Use the toggle to **activate** the public URL.
5. Click **Copy URL** to share it.

## Governance checklist (recommended)

Before sharing the link:
- Decide who is allowed to submit (and through which channel you will distribute the URL).
- Decide how you will handle **duplicates** (keep latest / keep best / manual review).
- Decide what “minimum quality” means (required fields, required documents, evidence).
- Decide when you will disable the link (typically right after the deadline).

## Monitor public submissions

1. From **Manage Assignments**, use:
   - **View All Public Submissions**, or
   - **View Submissions** for a specific assignment.
2. Review each submission for completeness and quality.
3. Use available actions (for example approve/reject) to manage quality control.

## Best practices

- Share the URL only through trusted channels (email lists you control, official platforms).
- Add clear instructions in the assignment message (what to fill, deadline, who to contact).
- Monitor submissions regularly during the collection period.
- Decide how to handle duplicates (for example: keep the most recent, or keep the one with the best sources).
- Disable the public URL when the collection window ends (to reduce risk).
- If you suspect the link was shared too broadly, disable it immediately and create a new controlled channel.

## Common problems

- **Public URL “doesn’t work”**: confirm it is activated and the assignment is still active.
- **Too many incorrect submissions**: add clearer instructions, reduce ambiguity, and consider disabling the public URL.
- **Duplicate submissions**: decide your rule (keep latest / keep best / manual merge) and apply consistently.
