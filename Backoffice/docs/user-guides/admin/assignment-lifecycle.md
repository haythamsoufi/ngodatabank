# Assignment lifecycle (from planning to closure)

Use this guide to manage an assignment over time: before it starts, during collection, and after submission.

## Before you start

- You need **Admin** access.
- Confirm the template is ready and tested.
- Decide the timeline (start date, deadline, review window).

## Plan (before launch)

- Choose a clear **period name** (so countries recognize it).
- Write a short instruction message:
  - What is requested
  - Deadline
  - Who to contact for questions
- Decide whether to use a **public URL** (only if needed).

## Launch

1. Create the assignment.
2. Confirm the right countries are included.
3. Confirm focal points for those countries have:
   - the correct assignment roles, and
   - the correct country access.

## Monitor during collection

During the open period, check progress regularly:

- Countries not started
- Countries in progress
- Countries submitted
- Overdue countries

If many countries struggle:

- Extend the deadline (if allowed in your workflow).
- Clarify confusing questions in the template (label/help text).

## Handle changes after launch

Typical changes include:

- **Add countries**: add them in the assignment edit screen, then notify focal points.
- **Extend deadline**: communicate the new deadline clearly.
- **Template changes**: avoid large template changes mid-collection; if needed, publish a new version and create a new assignment (preferred).

## Review and close

After submissions arrive:

- Review for quality (consistency, outliers, missing required fields).
- Approve or reopen submissions as needed.
- Export data for reporting.

## Common problems

- **A focal point can’t see the assignment**: check their roles and country access first.
- **Low submission rate**: consider reminders, deadline changes, and simpler templates.
- **Confusing template fields**: fix labels/help text and reduce required fields.
