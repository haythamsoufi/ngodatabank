# Data handling and privacy (basic guidance)

Use this page to reduce risk when collecting, exporting, and sharing data.

## What to avoid in submissions

Unless explicitly requested and approved by your process:
- avoid personal identifiers (names, phone numbers, addresses)
- avoid attaching documents with sensitive personal data

If sensitive data is required, agree on:
- who can access it
- how long it is retained
- how it is shared securely

## Supporting documents (evidence files)

Good practices:
- use clear file names (example: `Country_Period_Source.pdf`)
- upload the minimum necessary
- remove personal identifiers from documents when possible

See: [Supporting documents (Focal point)](../focal-point/supporting-documents.md).

## Export files (Excel/CSV)

Treat exports as sensitive:
- do not share via public links
- store in approved locations (shared drives with access control)
- keep a “raw export” copy unchanged
- document any manual cleaning steps

## Public URL submissions (special risk)

Public URLs can be shared widely.

If you use them:
- share only through trusted channels
- monitor submissions regularly
- disable the link when the collection period ends

If your process uses public links, follow your organization’s guidance. Ask your administrator how public URL submissions are set up and when links should be disabled.
