# Troubleshooting basics (everyone)

Use this page for quick fixes before escalating to an admin/support team.

## First checks (fast)

- Refresh the page.
- Log out and log back in.
- Try another browser (if possible).
- If you are on a slow connection, wait a few seconds after clicking buttons before closing the tab.

## “Access denied” / “You don’t have permission”

This usually means one of:
- your role does not allow that page/action
- you do not have access to the country/organization for that assignment

What to do:
- Contact your admin and share:
  - the page you tried to open
  - the assignment name (if relevant)
  - a screenshot of the message

## Buttons missing (Edit / Submit / Approve)

This is usually caused by:
- the submission status (submitted/approved can be locked), or
- missing permissions (role not assigned)

See: [Submission statuses and what you can do](submission-statuses-and-permissions.md)

## Saving issues (progress missing)

If you think answers were not saved:
- Look for a “Save draft” button (if present) and use it before leaving.
- Avoid closing the tab right after making many changes.
- If the page froze, refresh once and check if your answers are still there.

When contacting support, share:
- assignment name
- approximate time you entered the data
- what sections were affected

## File upload issues (supporting documents)

Common causes:
- file is too large
- file type is not allowed
- your role does not allow document upload

See: [Supporting documents (Focal point)](../focal-point/supporting-documents.md).
