# Gestion des données et confidentialité (conseils de base)

Utilisez cette page pour réduire les risques lors de la collecte, de l'exportation et du partage de données.

## Ce qu'il faut éviter dans les soumissions

Sauf demande explicite et approbation de votre processus :
- évitez les identifiants personnels (noms, numéros de téléphone, adresses)
- évitez de joindre des documents contenant des données personnelles sensibles

Si des données sensibles sont requises, convenez de :
- qui peut y accéder
- combien de temps elles sont conservées
- comment elles sont partagées en toute sécurité

## Documents justificatifs (fichiers de preuve)

Bonnes pratiques :
- utilisez des noms de fichiers clairs (exemple : `Country_Period_Source.pdf`)
- téléchargez le minimum nécessaire
- supprimez les identifiants personnels des documents lorsque c'est possible

Voir : [Documents justificatifs (Point focal)](../focal-point/supporting-documents.md).

## Fichiers d'export (Excel/CSV)

Traitez les exports comme sensibles :
- ne partagez pas via des liens publics
- stockez dans des emplacements approuvés (disques partagés avec contrôle d'accès)
- gardez une copie "export brut" inchangée
- documentez toutes les étapes de nettoyage manuel

## Soumissions d'URL publiques (risque spécial)

Les URL publiques peuvent être largement partagées.

Si vous les utilisez :
- partagez uniquement via des canaux de confiance
- surveillez les soumissions régulièrement
- désactivez le lien lorsque la période de collecte se termine

Si votre processus utilise des liens publics, suivez les consignes de votre organisation. Demandez à votre administrateur comment les soumissions par URL publique sont configurées et quand les liens doivent être désactivés.
