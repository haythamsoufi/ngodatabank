# How the platform works

This page explains the main pieces of the Backoffice in simple terms.

## Key ideas (plain language)

- **Template**: the form structure (questions + sections) that admins create.
- **Assignment**: when an admin sends a template to one or more countries for completion.
- **Form submission**: when a focal point finishes the form and clicks **Submit**.

## Typical workflow

1. An **admin** creates or updates a **template**.
2. The admin creates an **assignment** using that template and selects the country/countries.
3. A **focal point** opens the assignment, fills in the form, fixes validation messages, and clicks **Submit**.
4. Admins can monitor progress (for example: pending / in progress / submitted).
