# Glossaire

Définitions courtes des termes courants utilisés dans le Backoffice.

## Mission

Une **mission** est une demande de compléter un formulaire pour un pays spécifique (ou plusieurs pays). Les missions ont généralement une date limite et un statut.

## Modèle

Un **modèle** est la structure d'un formulaire (sections et questions) créée par les administrateurs. Les modèles sont utilisés pour créer des missions.

## Point focal

Un **point focal** est un utilisateur qui remplit et soumet les formulaires assignés (généralement pour un ou plusieurs pays).

## Administrateur

Un **administrateur** est un utilisateur qui gère les paramètres comme les utilisateurs, les modèles et les missions.

## Accès au pays

L'**accès au pays** contrôle les pays pour lesquels un utilisateur peut voir ou soumettre des données. Si un point focal n'a pas de pays assignés, il ne peut pas voir les missions pour ces pays.

## Champ requis

Un **champ requis** doit être rempli avant de pouvoir soumettre un formulaire. Les champs requis sont généralement marqués d'un astérisque (*).

## Message de validation

Un **message de validation** est un avertissement ou une erreur qui vous indique qu'il manque quelque chose ou qu'il y a une erreur dans le formulaire (par exemple : un champ requis est vide).

## Soumettre

**Soumettre** finalise vos données pour cette mission. Après la soumission, vous ne pourrez peut-être pas modifier le formulaire (selon le flux de travail et les permissions).

## Enregistrer brouillon / sauvegarde automatique

De nombreux formulaires **s'enregistrent automatiquement** pendant que vous travaillez. Certains formulaires peuvent également afficher un bouton **Enregistrer brouillon**.
