# Glossary

Short definitions of common terms used in the Backoffice.

## Assignment

An **assignment** is a request to complete a form for a specific country (or multiple countries). Assignments usually have a deadline and a status.

## Template

A **template** is the structure of a form (sections and questions) created by admins. Templates are used to create assignments.

## Focal point

A **focal point** is a user who fills in and submits assigned forms (usually for one or more countries).

## Admin

An **admin** is a user who manages settings like users, templates, and assignments.

## Country access

**Country access** controls which countries a user can view or submit data for. If a focal point has no assigned countries, they cannot see assignments for those countries.

## Required field

A **required field** must be filled in before you can submit a form. Required fields are usually marked with an asterisk (*).

## Validation message

A **validation message** is a warning or error that tells you something is missing or incorrect in the form (for example: a required field is empty).

## Submit

**Submit** finalizes your data for that assignment. After submission, you may not be able to edit the form (depending on the workflow and permissions).

## Save draft / autosave

Many forms **save automatically** as you work. Some forms may also show a **Save draft** button.
