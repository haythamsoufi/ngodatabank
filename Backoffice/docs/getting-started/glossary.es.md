# Glosario

Definiciones breves de términos comunes utilizados en el Backoffice.

## Asignación

Una **asignación** es una solicitud para completar un formulario para un país específico (o varios países). Las asignaciones suelen tener una fecha límite y un estado.

## Plantilla

Una **plantilla** es la estructura de un formulario (secciones y preguntas) creada por administradores. Las plantillas se utilizan para crear asignaciones.

## Punto focal

Un **punto focal** es un usuario que completa y envía formularios asignados (generalmente para uno o más países).

## Administrador

Un **administrador** es un usuario que gestiona configuraciones como usuarios, plantillas y asignaciones.

## Acceso al país

El **acceso al país** controla para qué países un usuario puede ver o enviar datos. Si un punto focal no tiene países asignados, no puede ver asignaciones para esos países.

## Campo requerido

Un **campo requerido** debe completarse antes de poder enviar un formulario. Los campos requeridos generalmente se marcan con un asterisco (*).

## Mensaje de validación

Un **mensaje de validación** es una advertencia o error que te indica que algo falta o es incorrecto en el formulario (por ejemplo: un campo requerido está vacío).

## Enviar

**Enviar** finaliza tus datos para esa asignación. Después del envío, es posible que no puedas editar el formulario (dependiendo del flujo de trabajo y los permisos).

## Guardar borrador / guardado automático

Muchos formularios se **guardan automáticamente** mientras trabajas. Algunos formularios también pueden mostrar un botón **Guardar borrador**.
